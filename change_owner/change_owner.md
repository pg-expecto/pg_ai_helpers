# Скрипт смены владельца объектов PostgreSQL

Данный скрипт (`change_owner.sql`) предназначен для массовой смены владельца всех объектов базы данных PostgreSQL на заданного пользователя. Он обрабатывает схемы, таблицы, представления, материализованные представления, последовательности, а также функции и процедуры.

## Назначение

Скрипт позволяет быстро передать права владения всеми объектами базы данных новому пользователю (например, при миграции или смене служебной учётной записи). Он автоматически обходит системные схемы и исключает определённые объекты, которые не должны менять владельца.

## Требования

- PostgreSQL 9.3+ (используется `pg_get_function_identity_arguments`)
- Пользователь, выполняющий скрипт, должен иметь права суперпользователя или быть владельцем объектов, для которых выполняется смена владельца (ALTER OWNER).
- В скрипте задан новый владелец по умолчанию — `poy_user`. При необходимости его нужно изменить в коде.

## Использование

1. Отредактируйте переменную `new_owner` в начале скрипта, указав имя пользователя, который станет новым владельцем:
   ```sql
   new_owner text := 'your_new_owner'; -- Укажите здесь имя нового владельца
   ```

2. Подключитесь к целевой базе данных (не к `postgres`, а к конкретной БД) и выполните скрипт:
   ```bash
   psql -d your_database -f change_owner.sql
   ```
   или в интерактивном режиме:
   ```sql
   \i change_owner.sql
   ```

## Что делает скрипт

### 1. Схемы
Меняет владельца для всех несистемных схем. Системные схемы (`pg_catalog`, `pg_toast`, `information_schema`, `public`) и схемы, начинающиеся с `pg_`, пропускаются.

### 2. Таблицы, представления, материализованные представления, последовательности
Обрабатываются все объекты в несистемных схемах, соответствующие типам:
- обычные таблицы (`TABLE`)
- представления (`VIEW`)
- материализованные представления (`MATERIALIZED VIEW`)
- последовательности (`SEQUENCE`)

### 3. Функции и процедуры
Для всех функций и процедур (включая перегруженные) выполняется смена владельца. Исключаются функции, имя которых содержит `dblink` (регистронезависимо) — они обычно относятся к расширению dblink и не должны менять владельца.

## Предостережения

- Скрипт не обрабатывает владельца баз данных, типов данных, операторов и других объектов. При необходимости их нужно менять отдельно.
- Если в базе есть объекты в схемах, которые начинаются с `pg_` и не являются системными (например, схемы расширений), они не будут обработаны. Это сделано намеренно, чтобы не нарушить работу расширений.
- Для функций скрипт использует `pg_get_function_identity_arguments`, что позволяет корректно различать перегруженные функции.
- Скрипт не выдаёт предварительный список изменений; все операции выполняются сразу. Рекомендуется предварительно протестировать на копии базы данных.

## Пример

Допустим, нужно передать владение всеми объектами в базе `mydb` пользователю `app_user`. Отредактируйте скрипт:

```sql
new_owner text := 'app_user';
```

Затем выполните:

```bash
psql -d mydb -f change_owner.sql
```

После выполнения все объекты (схемы, таблицы, представления, последовательности, функции) станут принадлежать `app_user`.

## Примечания

- Если требуется изменить владельца также для баз данных или расширений, используйте отдельные команды `ALTER DATABASE ... OWNER TO` и `ALTER EXTENSION ... OWNER TO`.
- В случае ошибки выполнения (например, отсутствие прав) скрипт остановится. Для продолжения можно обернуть выполнение в транзакцию и выполнять с `\set ON_ERROR_STOP off`, но это может привести к частичному изменению владельцев.

## Автор
Скрипт создан для автоматизации смены владельца объектов PostgreSQL. При необходимости адаптируйте под свою среду.
